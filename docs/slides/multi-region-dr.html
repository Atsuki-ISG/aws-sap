Backup & Restore

S3にバックアップ保存。障害時にAMI/スナップショットから復元。最低コストだが復旧に時間。

Pilot Light

DBレプリカなどコア部分のみ常時稼働。障害時にEC2/ECSをスケールアウト。中コスト。

Warm Standby

本番の縮小版を常時稼働。障害時にスケールアップ。素早い復旧。やや高コスト。

Multi-Site Active-Active

全リージョンで本番トラフィック処理。DNS加重でルーティング。最高コスト・即時復旧。

SAP頻出ポイント
Pilot Light と Warm Standby の違い: Pilot Light はDBのみ稼働、Warm Standbyはアプリも縮小稼働。
「最小コストで数時間以内のRTO」→ Pilot Light。「分単位のRTO」→ Warm Standby。

静的コンテンツ (S3)

S3 クロスリージョンレプリケーション (CRR)
S3バケットを別リージョンに非同期レプリケーション。CloudFrontのオリジンフェイルオーバーでPrimary S3障害時にDR側S3へ自動切替。
S3 Transfer Accelerationでアップロードも高速化可能。

動的コンテンツ (ALB + Aurora)

CloudFront Origin Failover
CloudFrontのOrigin Groupを設定し、Primary OriginとSecondary Originを定義。
5xx/4xxエラー時に自動でSecondary Originへフォールバック。
注意: Origin FailoverはHTTPエラーコードベース。ネットワークレベルの障害にも対応。
ひっかけ注意
CloudFrontのOrigin Failoverは読取り(GET/HEAD)のみ。POST/PUTにはフェイルオーバーしない。
書込みのDRにはRoute 53フェイルオーバーを別途構成する必要がある。

結果整合性 vs 強い整合性

強い整合性
読取りが常に最新の書込み結果を返す。DynamoDBのConsistentRead=true、S3のPUT後のGET（2020年以降）
結果整合性
一定時間後に全レプリカが最新化。DynamoDB Global Tablesのクロスリージョン、S3 CRR、Aurora Global DBの読取り
DR設計での判断ポイント
同期レプリケーションはクロスリージョンに不向き（ネットワークレイテンシ）。
クロスリージョンDRでは非同期が前提 → RPO > 0 を受容する設計が必要。
RPO = 0 が必須なら → 同一リージョン内Multi-AZ構成 + リージョン障害は許容。
サービス別の整合性まとめ
Aurora Global DB: リードレプリカは結果整合性。フェイルオーバー後は強い整合性に復帰。
DynamoDB Global Tables: クロスリージョンは結果整合性。同一リージョン内は強い整合性を選択可。
S3: 同一リージョン内は強い整合性（read-after-write）。CRRは結果整合性。

ひっかけパターン & 練習問題

よくあるひっかけ 5選

1. 「Pilot LightでRTO 5分」
Pilot LightはDB以外のインフラ起動が必要。分単位のRTOならWarm Standby。
2. 「RDS Read ReplicaでマルチリージョンHA」
Read Replicaは読取専用。フェイルオーバー自動切替なし。DRにはAurora Global DB。
3. 「CloudFront Origin FailoverでPOSTをDRへ」
Origin FailoverはGET/HEADのみ。書込みDRにはRoute 53を使う。
4. 「DynamoDB Global TablesでRPO = 0」
Global Tablesは非同期レプリケーション。RPOはゼロではなく通常1秒以内。
5. 「S3 CRRでリアルタイム同期」
CRRは非同期。15分以内に99.99%が複製されるがリアルタイムではない。S3 Replication Time Control (RTC)で15分SLA。

練習問題 1

グローバルECサイト。要件: (1) リージョン障害時にRTO 1分以下 (2) RPO 1秒以下 (3) 読取りは各リージョンで低レイテンシ。データベース構成として最適なものは？
正解: B
A: Read Replicaの昇格は手動で時間がかかる。RTO 1分以下は困難。
B: Aurora Global DBはRPO 1秒以下、フェイルオーバー1分以下。各リージョンで読取り可能。
C: DynamoDB Global Tablesも候補だが、「ECサイト」で複雑なクエリ → リレーショナルDBが適切。Last Writer Winsの競合リスクあり。
D: RDS Multi-AZは同一リージョン内のHA。クロスリージョンDRには別途構成が必要。

練習問題 2

金融機関のWebアプリ。要件: (1) プライマリリージョン障害時にダウンタイムなし (2) 両リージョンで書込み可能 (3) コスト最適化より可用性優先。最適なDR構成は？
正解: C
A: Active-Passiveでは切替時にダウンタイムが発生。「ダウンタイムなし」を満たさない。
B: RDS Multi-AZは同一リージョン内のみ。CloudFront Origin FailoverはGET/HEADのみ。
C: DynamoDB Global TablesはActive-Active Multi-Master。両リージョンで書込み可能。サーバーレスで両リージョン完全稼働。
D: Warm StandbyはActive-Passive。障害時にスケールアップが必要でダウンタイムが発生。
